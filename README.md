# WebVoxel 3D

Браузерная 3D voxel-песочница: процедурный блочный мир, вид от первого лица, разрушение и установка блоков, сохранение изменений на сервере.

**Стек:** Node.js + TypeScript + Express + Socket.IO (сервер) · Vite + TypeScript + Three.js (клиент)

## Требования

- Node.js ≥ 18 (npm ≥ 9)

## Установка

```bash
npm install
```

Одна команда ставит зависимости для всех workspaces (корень, `server`, `client`).

## Запуск в dev-режиме

```bash
npm run dev
```

Поднимает одновременно:

- **сервер** на `http://localhost:3000` (tsx watch — авторестарт при изменениях),
- **клиент** на `http://localhost:5173` (Vite с HMR; запросы `/api` и `/socket.io` проксируются на сервер).

Открой **http://localhost:5173** в браузере. Можно запускать и по отдельности: `npm run dev:server`, `npm run dev:client`.

## Продакшн-сборка

```bash
npm run build   # собирает клиент (dist) и сервер (dist)
npm run start   # запускает сервер, который отдаёт и API, и собранный клиент
```

Открой **http://localhost:3000**.

Переменные окружения сервера:

- `PORT` — порт (по умолчанию `3000`);
- `WORLD_FILE` — путь к файлу мира (по умолчанию `server/data/world.json`).

## Управление

| Клавиша | Действие |
| --- | --- |
| **W A S D** | движение |
| **Space** | прыжок |
| **Мышь** | обзор (Pointer Lock — кликни по экрану, Esc — отпустить мышь) |
| **ЛКМ** | разрушить блок (наведённый блок подсвечивается рамкой) |
| **ПКМ** | поставить блок на соседнюю грань |
| **1–4** | выбор блока: Grass / Dirt / Stone / Wood |

HUD показывает FPS, игровое время (☀/☾ ЧЧ:ММ), координаты игрока и выбранный блок.

### Меню, миры и настройки

При запуске открывается **главное меню**. В разделе «Одиночная игра» — список
сохранённых миров (имя, сид, время последней игры) с кнопками «Играть» и
«Удалить», а также «Создать новый мир» (имя + необязательный сид — число или
текст; пусто = случайный). Каждый мир хранится на сервере отдельным файлом и
переживает перезапуск.

**Esc** в игре открывает паузу: «Продолжить», «Настройки», «Выйти в меню».

**Настройки** (сохраняются в браузере, применяются на лету): дальность
прорисовки, поле зрения (FOV), чувствительность мыши, длина дня.

### Цикл дня и ночи

Мир живёт по циклу **20 минут** (10 минут день + 10 минут ночь). Солнце и луна
движутся по небу; вместе с ними меняются цвет неба и тумана, сила и цвет
освещения (тёплое солнце днём, холодная луна ночью) и на закате/рассвете небо
становится оранжевым. Блоки отбрасывают тени (shadow mapping) — источник света
следует за солнцем/луной, а его теневая камера едет за игроком, чтобы тени
оставались чёткими в пределах прорисовки.

## Архитектура

```
/shared                  — общий код клиента и сервера (единственный источник правды)
  blocks.ts              — типы блоков (air/grass/dirt/stone/wood), цвета, твёрдость
  constants.ts           — размер чанка 16×16×64, раскладка индексов блоков
  protocol.ts            — типы сообщений REST/Socket.IO

/server/src
  index.ts               — Express + Socket.IO + статика клиента в продакшне
  world/generator.ts     — детерминированная генерация: 2 октавы value noise,
                           grass сверху, 3 слоя dirt, ниже stone
  world/worldStore.ts    — мир = генерация + правки игрока поверх;
                           в JSON сохраняются только правки (замена на SQL — в одном классе)
  world/worldManager.ts  — реестр миров: data/worlds/<id>.json + index.json,
                           создание/удаление/ленивое открытие store'ов
  world/chunk.ts         — серверный чанк
  world/blockTypes.ts    — реэкспорт shared-блоков
  api/worldsRoutes.ts    — GET/POST /api/worlds, DELETE /api/worlds/:id,
                           GET /api/worlds/:id/meta|chunk, POST /api/worlds/:id/block
  socket/socketServer.ts — игрок входит в комнату своего мира (worldId в handshake);
                           block:set → block:update и player:move — в пределах комнаты

/client
  public/textures/       — PNG-текстуры блоков (grass_top, grass_side, dirt,
                           stone, wood_top, wood_side); Vite копирует их в dist
/client/src
  main.ts                — точка входа
  App.ts                 — переключение меню ↔ игра, общий Settings, dispose сессий
  ui/Menu.ts             — меню: главное / список миров / создание / настройки
  ui/PauseOverlay.ts     — пауза в игре (продолжить / настройки / выйти)
  game/Settings.ts       — настройки (localStorage) + форма настроек
  game/WorldsApi.ts      — REST-клиент меню (список/создание/удаление миров)
  game/Game.ts           — сборка подсистем + главный цикл; dispose() чистит сессию
  game/Renderer.ts       — WebGL, сцена, камера, свет (directional + ambient), туман
  game/World.ts          — потоковый стриминг чанков вокруг игрока (радиус 16,
                           ближние первыми, лимит загрузок/ремешей на кадр), block lookup,
                           планирование ремешей
  game/Chunk.ts          — данные чанка + его mesh
  game/ChunkMesher.ts    — один BufferGeometry на чанк; air не рендерится,
                           скрытые грани отсекаются (в т.ч. на границах чанков);
                           грани группируются по материалам (per-face текстуры)
  game/Textures.ts       — загрузка текстур блоков, per-face маппинг,
                           NearestFilter + мипмапы, иконки хотбара
  game/Sky.ts            — цикл день/ночь (10 мин день + 10 мин ночь),
                           солнце и луна, динамическое небо/туман/свет,
                           тени от блоков (directional light + shadow map)
  game/Player.ts         — AABB-физика: гравитация, прыжок, поосевые коллизии
  game/CameraController.ts — yaw/pitch от первого лица
  game/Raycaster.ts      — воксельный DDA (Amanatides & Woo) для выбора блока
  game/Input.ts          — клавиатура, мышь, Pointer Lock
  game/HUD.ts            — прицел, FPS, время, координаты, хотбар
  game/Network.ts        — REST-загрузка мира (по worldId) + Socket.IO-события
  game/BlockRegistry.ts  — реэкспорт shared-блоков + цвета для Three.js
```

### Ключевые решения

- **Мир = генерация + diff.** Сервер хранит только изменённые игроком блоки
  (`server/data/worlds/<id>.json`); базовый рельеф всегда регенерируется из seed.
  Чанк в API — это генерация с наложенными правками.
- **Много миров, изоляция по комнатам.** `WorldManager` ведёт реестр миров
  (`index.json`) и открывает store'ы лениво. Каждый сокет входит в комнату
  своего мира, поэтому правки и позиции не текут между мирами. Удаление мира
  закрывает store (правки после закрытия отклоняются — файл не воскресает) и
  выкидывает подключённых игроков в меню.
- **Один меш на чанк.** Никаких мешей на блок: чанк собирается в один
  `BufferGeometry` с UV; грани группируются по материалам (по одной текстуре на
  материал), поэтому чанк рисуется в несколько draw-call'ов. При изменении блока
  пересобирается только затронутый чанк (и сосед, если блок на границе).
- **Правки идут через сервер.** Клиент применяет правку локально и шлёт
  `block:set`; сервер валидирует, сохраняет и рассылает `block:update` всем.
  Эхо собственной правки — идемпотентный no-op, чужие правки применяются так же:
  это и есть заготовка мультиплеера.
- **Своя физика.** Гравитация + поосевое разрешение AABB-коллизий с сеткой
  блоков, с сабстепами против туннелирования. Никакого физического движка.

### Что заложено на будущее

Мультиплеер (позиции игроков уже ходят по сокету в пределах комнаты мира),
инвентарь/крафт (hotbar — отдельный модуль), биомы (генератор изолирован), БД
вместо JSON (весь I/O — в `WorldStore`/`WorldManager`), мобы. Текстуры,
динамическое освещение, цикл дня/ночи с тенями, меню, сохранение нескольких
миров и настройки уже реализованы.
